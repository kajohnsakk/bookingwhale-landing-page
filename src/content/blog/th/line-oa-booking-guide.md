คนไทย **54 ล้านคน** ใช้ LINE ทุกวัน และ 93% ของผู้ใช้สมาร์ทโฟนในไทยมีแอป LINE ติดตั้งอยู่ในเครื่อง (LINE Thailand Official, 2025)

ถ้าธุรกิจของคุณยังให้ลูกค้าจองผ่านโทรศัพท์หรือ LINE ส่วนตัว คุณกำลังพลาดโอกาสมหาศาล เพราะลูกค้าอยู่บน LINE อยู่แล้ว แค่ต้องให้พวกเขาจองได้จากในนั้นเลย

คู่มือนี้จะพาคุณตั้งค่าระบบจองคิวผ่าน LINE OA แบบทีละขั้นตอน ตั้งแต่สร้างบัญชีไปจนถึงแจ้งเตือนอัตโนมัติ

## ทำไม LINE OA ถึงสำคัญที่สุดสำหรับธุรกิจไทย?

### ตัวเลขที่พูดแทนทุกอย่าง

- **93%** ของผู้ใช้สมาร์ทโฟนในไทยใช้ LINE
- **85%** ของข้อความ LINE OA ถูกเปิดอ่านภายใน 1 ชั่วโมง
- ลูกค้าที่ติดตาม LINE OA มีโอกาสซื้อซ้ำสูงกว่าช่องทางอื่น **3.2 เท่า**
- **67%** ของลูกค้าไทยชอบติดต่อธุรกิจผ่าน LINE มากกว่าโทรศัพท์ (Kantar Thailand, 2025)

### ข้อได้เปรียบของ LINE OA เทียบกับช่องทางอื่น

| ช่องทาง | อัตราเปิดอ่าน | ตอบโต้ได้ | ส่งไฟล์/รูป | ค่าใช้จ่าย |
|---|---|---|---|---|
| LINE OA | 85% | ใช่ | ใช่ | ต่ำ |
| Email | 20-25% | จำกัด | ใช่ | ต่ำ |
| SMS | 45% | ไม่ | ไม่ | ปานกลาง |
| โทรศัพท์ | 100% | ใช่ | ไม่ | สูง |

LINE OA ชนะในเกือบทุกมิติสำหรับตลาดไทย

## LINE OA มีกี่ประเภท? เลือกแบบไหนดี?

ก่อนเริ่มต้น ต้องเข้าใจประเภทของ LINE OA

### LINE Official Account (OA)

เหมาะสำหรับธุรกิจทุกขนาด สามารถส่งข้อความ Broadcast, ตั้ง Rich Menu, ใช้ Messaging API ได้ มีแพ็กเกจฟรีและแบบเสียเงิน

**แพ็กเกจที่แนะนำสำหรับธุรกิจบริการ**

- **Free Plan** — ส่งข้อความฟรี 500 ครั้ง/เดือน เหมาะสำหรับร้านขนาดเล็กที่เพิ่งเริ่ม
- **Light Plan** — 1,200 บาท/เดือน ส่งได้ 5,000 ครั้ง เหมาะสำหรับร้านที่มีลูกค้า 100-500 คน
- **Standard Plan** — 3,000 บาท/เดือน ส่งได้ไม่จำกัด เหมาะสำหรับร้านขนาดกลางขึ้นไป

## ขั้นตอนที่ 1: สร้าง LINE Official Account

### 1.1 ลงทะเบียน LINE OA

1. ไปที่ [LINE Official Account Manager](https://manager.line.biz/)
2. ล็อกอินด้วย LINE ID หรือสร้างบัญชีธุรกิจใหม่
3. กดปุ่ม "สร้าง Official Account"
4. กรอกข้อมูลธุรกิจ: ชื่อ, ประเภทธุรกิจ, เบอร์โทร, อีเมล
5. อัปโหลดโลโก้และรูปปกที่สื่อถึงแบรนด์ชัดเจน
6. รอการอนุมัติ (โดยทั่วไป 1-3 วันทำการ)

### 1.2 ตั้งค่าพื้นฐาน

หลังได้รับการอนุมัติ ตั้งค่าเหล่านี้ก่อน

- **ชื่อ OA** — ควรตรงกับชื่อร้านหรือแบรนด์ที่ลูกค้ารู้จัก
- **คำทักทายอัตโนมัติ** — ข้อความที่ส่งทันทีเมื่อลูกค้า Add Friend
- **ข้อความตอบกลับอัตโนมัติ** — สำหรับนอกเวลาทำการ

**ตัวอย่างข้อความทักทาย**

> "ยินดีต้อนรับสู่ [ชื่อร้าน] นะคะ!
> กดเมนูด้านล่างเพื่อจองคิว ดูโปรโมชัน หรือติดต่อทีมงานได้เลยค่ะ"

## ขั้นตอนที่ 2: สร้าง Rich Menu สำหรับการจองคิว

Rich Menu คือเมนูที่ปรากฏที่ด้านล่างของหน้าแชท ลูกค้ากดได้ทันทีโดยไม่ต้องพิมพ์อะไร

### 2.1 ออกแบบ Rich Menu

ขนาดที่แนะนำ: **2500 x 1686 พิกเซล** (สำหรับ Rich Menu เต็ม) หรือ **2500 x 843 พิกเซล** (ครึ่งหน้า)

เนื้อหาที่ควรมีใน Rich Menu สำหรับธุรกิจจองคิว

| พื้นที่ | ข้อความ | การกระทำ |
|---|---|---|
| ปุ่มใหญ่ตรงกลาง | "จองคิว" | เปิดหน้าจองออนไลน์ |
| บนซ้าย | "ดูนัดหมาย" | เปิดหน้าประวัติการจอง |
| บนขวา | "โปรโมชัน" | เปิดหน้าโปรโมชัน |
| ล่างซ้าย | "ติดต่อเรา" | เปิดแชท |
| ล่างขวา | "เกี่ยวกับเรา" | เปิดเว็บไซต์ |

### 2.2 อัปโหลด Rich Menu ใน LINE OA Manager

1. ไปที่ "Rich Menu" ในเมนูซ้าย
2. กด "สร้าง Rich Menu ใหม่"
3. อัปโหลดภาพที่ออกแบบไว้
4. ตั้งค่า Action สำหรับแต่ละพื้นที่
5. สำหรับปุ่มจองคิว ตั้ง Action เป็น "URI" และใส่ลิงก์จองของ Booking Whale
6. กำหนดช่วงเวลาที่ต้องการให้แสดง Rich Menu
7. กดเผยแพร่

### 2.3 เชื่อมลิงก์จอง Booking Whale กับ Rich Menu

ลิงก์จองของ Booking Whale สามารถใส่ตรงใน URI ของปุ่ม Rich Menu ได้เลย เมื่อลูกค้ากดปุ่ม "จองคิว" ในแชท LINE จะเปิดหน้าจองอัตโนมัติใน LINE browser

## ขั้นตอนที่ 3: เชื่อมต่อ Booking Whale กับ LINE OA

### 3.1 เปิดใช้งาน Messaging API

1. ในหน้า LINE OA Manager ไปที่ "Settings" > "Messaging API"
2. กด "Enable Messaging API"
3. เลือก LINE Developers Provider หรือสร้างใหม่
4. คัดลอก **Channel ID, Channel Secret** และ **Channel Access Token**

### 3.2 เชื่อมต่อใน Booking Whale

1. เข้าสู่ระบบ Booking Whale
2. ไปที่ "การตั้งค่า" > "การเชื่อมต่อ" > "LINE OA"
3. วาง Channel ID, Channel Secret และ Channel Access Token
4. กด "ทดสอบการเชื่อมต่อ"
5. คัดลอก Webhook URL ที่ระบบสร้างให้
6. กลับไปที่ LINE OA Manager และวาง Webhook URL ในช่อง Webhook URL
7. กด "Verify" เพื่อยืนยัน

ดูฟีเจอร์การเชื่อมต่อทั้งหมดได้ที่ [หน้าฟีเจอร์ Booking Whale](/\#features)

### 3.3 ทดสอบการเชื่อมต่อ

ส่ง LINE ทดสอบโดยการจองนัดผ่านหน้าจองของ Booking Whale แล้วตรวจสอบว่าได้รับข้อความยืนยันใน LINE OA ของคุณ (ในฐานะเจ้าของร้าน) และลูกค้าได้รับข้อความยืนยันที่ LINE ของพวกเขา

## ขั้นตอนที่ 4: ตั้งค่า Auto-Reply สำหรับการจอง

Auto-Reply ที่ดีช่วยให้ลูกค้าได้รับข้อมูลที่ต้องการทันทีโดยไม่ต้องรอพนักงาน

### 4.1 Keyword Auto-Reply

ตั้งค่าตอบกลับอัตโนมัติสำหรับ keyword สำคัญ

| Keyword | ข้อความตอบกลับ |
|---|---|
| "จอง" | ส่งลิงก์หน้าจองพร้อมข้อมูลบริการ |
| "ราคา" | ส่งรายการราคาบริการ |
| "เปิด" หรือ "เวลา" | ส่งเวลาทำการ |
| "ที่อยู่" | ส่งแผนที่และที่อยู่ |
| "โปรโมชัน" | ส่งรายการโปรโมชันปัจจุบัน |

### 4.2 ข้อความนอกเวลาทำการ

ตั้งค่าข้อความอัตโนมัติสำหรับนอกเวลาทำการ

## Best Practices สำหรับ LINE OA ธุรกิจบริการ

### สิ่งที่ควรทำ

- **ส่ง Broadcast เดือนละ 2-4 ครั้ง** — โปรโมชัน เคล็ดลับ หรือข่าวสารที่มีประโยชน์ต่อลูกค้า
- **ใช้รูปภาพในการแจ้งเตือน** — ข้อความพร้อมรูปได้รับการตอบสนองสูงกว่าข้อความล้วน 40%
- **ตอบกลับภายใน 1 ชั่วโมง** — ในเวลาทำการ การตอบช้าทำให้ลูกค้าไปหาคู่แข่ง
- **อัปเดต Rich Menu ตามฤดูกาล** — เปลี่ยน Rich Menu ตามโปรโมชันหรือช่วงเทศกาล
- **วัดผล** — ดูสถิติการเปิดอ่าน การคลิก และ Follower Growth ทุกเดือน

### สิ่งที่ไม่ควรทำ

- **ส่ง Broadcast มากเกินไป** — เกิน 8 ครั้ง/เดือนทำให้ลูกค้า Block
- **ข้อความที่ยาวเกินไป** — ตัดให้เหลือสั้นกระชับ ใจความสำคัญอยู่ใน 3 บรรทัดแรก
- **ใช้ภาษาทางการเกินไป** — LINE เป็นช่องทางสื่อสารส่วนตัว ใช้ภาษาอบอุ่นเป็นกันเอง
- **ลืมทดสอบก่อน Broadcast** — ทดสอบข้อความกับบัญชีตัวเองก่อนส่งจริงทุกครั้ง
- **ไม่มี Call-to-Action** — ทุกข้อความควรมีสิ่งที่ต้องการให้ลูกค้าทำต่อเสมอ

## ข้อผิดพลาดที่พบบ่อยและวิธีแก้

### ข้อผิดพลาดที่ 1: Rich Menu ไม่แสดงหน้าจองที่ถูกต้อง

**สาเหตุ:** URL ในปุ่มไม่ใช่ลิงก์จองของระบบหรือลิงก์หมดอายุ
**วิธีแก้:** ตรวจสอบ URL ทุกปุ่มใน Rich Menu และทดสอบก่อน Publish

### ข้อผิดพลาดที่ 2: ลูกค้าไม่ได้รับข้อความแจ้งเตือน

**สาเหตุ:** ลูกค้ายังไม่ได้ Add Friend LINE OA หรือ Block บัญชีไว้
**วิธีแก้:** ในขั้นตอนจอง ขอให้ลูกค้าสแกน QR Code Add Friend ก่อน และตรวจสอบว่าระบบมีเบอร์หรือ LINE ID ที่ถูกต้อง

### ข้อผิดพลาดที่ 3: Webhook ตอบสนองช้าหรือไม่ทำงาน

**สาเหตุ:** Server ของระบบจองมีปัญหาหรือ Webhook URL ผิด
**วิธีแก้:** ตรวจสอบ Webhook URL และทดสอบการเชื่อมต่อใหม่ในหน้าการตั้งค่า Booking Whale

### ข้อผิดพลาดที่ 4: ข้อความซ้ำหรือส่งผิดคน

**สาเหตุ:** ตั้งค่า Trigger การแจ้งเตือนซ้ำซ้อน
**วิธีแก้:** ตรวจสอบการตั้งค่าแจ้งเตือนในระบบว่าไม่มีการตั้งค่าซ้ำกัน

## วิธีวัดความสำเร็จของ LINE OA

### ตัวชี้วัดสำคัญที่ควรติดตาม

- **Follower Growth Rate** — จำนวน Follower ใหม่ต่อเดือน เป้าหมาย +10-15%/เดือน
- **Message Open Rate** — อัตราการเปิดอ่าน เป้าหมาย >70%
- **Booking Conversion Rate** — % ของคนที่กดลิงก์จองแล้วจองจริง เป้าหมาย >30%
- **No-Show Rate** — ควรลดลงหลังใช้ระบบแจ้งเตือน เป้าหมาย <10%
- **Re-booking Rate** — % ลูกค้าที่กลับมาจองซ้ำ เป้าหมาย >40%

ดูวิธีลด No-Show ด้วยระบบแจ้งเตือนได้อย่างละเอียดที่ [วิธีลดลูกค้า No-Show 80%](/blog/reduce-no-show-80-percent)

## คำถามที่พบบ่อย (FAQ)

### ต้องมีความรู้เทคนิคแค่ไหนในการตั้งค่า LINE OA กับระบบจอง?

ไม่จำเป็นต้องมีความรู้เทคนิคพิเศษ กระบวนการเชื่อมต่อ Booking Whale กับ LINE OA ออกแบบให้ทำได้เองโดยไม่ต้องเขียนโปรแกรม ขั้นตอนหลักคือการคัดลอก Token จาก LINE และวางใน Booking Whale เท่านั้น ใช้เวลาประมาณ 30-60 นาที

### LINE OA ฟรีใช้ร่วมกับ Booking Whale ได้ไหม?

ได้ แต่มีข้อจำกัดด้านจำนวนข้อความที่ส่งได้ต่อเดือน (500 ข้อความ/เดือนสำหรับ Free Plan) สำหรับธุรกิจที่มีลูกค้าหลายร้อยคน แนะนำอย่างน้อย Light Plan ของ LINE OA เพื่อให้การแจ้งเตือนครอบคลุมลูกค้าทุกคน

### ลูกค้าต้อง Add Friend LINE OA ก่อนถึงจะจองได้ไหม?

ไม่จำเป็นเสมอไป ลูกค้าสามารถจองผ่านลิงก์หรือ QR Code โดยไม่ต้อง Add Friend ก่อนได้ แต่หากต้องการรับการแจ้งเตือนผ่าน LINE จะต้อง Add Friend เพื่อให้ระบบส่งข้อความหาได้ แนะนำให้ขอให้ลูกค้า Add Friend ในขั้นตอนจองเพื่อประสบการณ์ที่ดีที่สุด

### Rich Menu ต้องอัปเดตบ่อยแค่ไหน?

ขั้นต่ำควรอัปเดต Rich Menu ทุก 3 เดือนหรือเมื่อมีการเปลี่ยนแปลงบริการสำคัญ ธุรกิจที่ active มักอัปเดตตามเทศกาลหรือโปรโมชัน เช่น สงกรานต์ ปีใหม่ เพื่อดึงดูดลูกค้าให้กลับมามีปฏิสัมพันธ์กับ LINE OA

### ถ้าลูกค้า Block LINE OA จะเกิดอะไรขึ้นกับการจองที่มีอยู่?

การจองที่มีอยู่ยังคงอยู่ในระบบและพนักงานยังเห็น แต่ระบบจะไม่สามารถส่งข้อความแจ้งเตือนไปยังลูกค้าคนนั้นได้อีก ระบบที่ดีจะแจ้งให้เจ้าของร้านทราบเมื่อลูกค้า Block เพื่อให้โทรศัพท์แจ้งเตือนแทน
